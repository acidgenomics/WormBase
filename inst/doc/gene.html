Gene annotations with the gene() function

Michael J. Steinbaugh

2016-10-03

The goal of the worminfo package is to quickly and simply provide correct, up-to-date annotation data for the entire C. elegans genome. The underlying annotations provided by the gene() function are automatically sourced from WormBase, Ensembl and PANTHER. These source files are saved at the data branch of the worminfo GitHub repository. WormBase is updated on a bi-monthly release schedule. If the source data appears outdated, file an update request on GitHub, and weâ€™ll promptly issue a rebuild.

gene() accepts 4 types of identifiers: gene, name, sequence or class. Output from the function is controlled using the select parameter. The function returns a tibble data.frame that can be manipulated using the dplyr package or the base::merge() function. We recommend using dplyr::left_join() to bind these annotations to other data.frames.

identifier examples

WormBase gene:

identifier <- c("WBGene00000898", "WBGene00000912", "WBGene00004804")
gene(identifier, format = "gene")

WBGene00000898 Y55D5A.5 daf-2 abnormal DAuer Formation
WBGene00000912 R13H8.1 daf-16 abnormal DAuer Formation
WBGene00004804 T19E7.2 skn-1 SKiNhead

Sequence, commonly the open reading frame (ORF):

identifier <- c("Y55D5A.5", "R13H8.1", "T19E7.2")
gene(identifier, format = "sequence")

When select is not declared, a simple result is returned:

  • gene
  • sequence
  • name
  • class

Additionally, special identifiers, report and NULL declarations are available. These shortcuts are useful for generating summaries and may be relevant to experiments involving large gene lists.
